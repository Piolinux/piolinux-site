O que é chmod?

chmod (change mode) é um comando do Linux/Unix usado para alterar as permissões de acesso de arquivos e diretórios.O uso incorreto desses comandos pode causar problemas, tornando o sistema inoperante. As permissões são definidas para:

  • Dono (usuário) — Qual indivíduo é responsável pela criação dos referidos arquivos?
  • Grupo (grupo) — grupo associado ao arquivo
  • Outros (outros) — todos os demais usuários

Em caso de dúvidas, digite "man chmod" no terminal para maiores esclarecimentos.

Cuidado: Permissões perigosas

Nâo utilize use chmod 777 (permissão total para todos) em arquivos ou diretórios sensíveis. Isso permitirá que qualquer usuário leia, escreva e execute, abrindo brechas de segurança e causando problemas graves.

Boas práticas de usabilidade para permissões:

Tipo Permissão Numérica Permissão Simbólica Quando usar
Arquivo comum 644 -rw-r--r-- Arquivos de configuração, textos, scripts que não precisam ser executados
Script executável 755 -rwxr-xr-x Scripts que você (dono) executa, e outros só leem/executam
Diretório 755 drwxr-xr-x Diretórios que todos precisam listar e acessar, mas só dono modifica
Arquivo sensível 600 -rw------- Chaves SSH, arquivos de senha, configurações privadas — só o dono acessa
Diretório privado 700 drwx------ Diretórios com dados sensíveis — só o dono entra

Como usar esta calculadora?

Indique as permissões desejadas para cada categoria, assinalando as opções correspondentes (leitura, escrita, execução). O sistema exibirá o valor numérico equivalente (por exemplo, 755).

Exemplos seguros da utilização: chmod 644 arquivo.conf → dono: leitura+escrita, grupo/outros: só leitura.
